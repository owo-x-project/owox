# Task 005: ホバー時ジッター・レイアウト崩れの解消

## 目的

ホバー時のレイアウトジッター、要素のガタつき、overflow による崩れを特定し修正する。

## 前提条件

- Phase 01 完了

## 作業内容

- 全画面のホバー操作でジッター発生箇所を洗い出し
- 原因パターンの特定と修正:
  - ボーダー追加/削除によるサイズ変化 → `outline` や `box-shadow` で代替
  - `display: none` ↔ `block` による reflow → `visibility` / `opacity` で代替
  - `transform` 未使用のサイズ変化 → GPU アクセラレーション (`will-change`, `transform`) 利用
  - テキスト折り返し変化 → 固定幅/`min-width` 設定
  - スクロールバー出現/消滅 → `overflow: overlay` や `scrollbar-gutter: stable` 利用
- 各修正後に画面全体のレイアウト影響を確認

## 完了条件

- 報告されているジッター箇所が解消されている
- ホバー操作で周辺要素が動かない

## 検証方法

- 全画面でホバー操作を繰り返し、ガタつきがないことを確認
- 高速なマウス移動でもちらつかないことを確認

## 依存関係

- Phase 01 完了

## 成果物

- 各 CSS ファイル (変更)
- 必要に応じてコンポーネントの構造修正
