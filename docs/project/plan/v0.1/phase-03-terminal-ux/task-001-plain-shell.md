# Task 001: プレーンシェル起動

## 目的

ターミナル新規作成時にコマンド指定なしでデフォルトシェル ($SHELL) を起動する。

## 前提条件

- バックエンドの `POST /terminal/sessions` API が `command` 省略時にデフォルトシェルを起動できる (要確認)

## 作業内容

- `CommandLauncher.tsx` のターミナル作成フローを変更:
  - 「新しいターミナル」ボタンを追加 (コマンド入力不要で即起動)
  - 既存のコマンド指定起動も維持
- `TerminalSurface.tsx` のセッション作成ロジック:
  - `command` を省略して `create()` を呼び出す
  - バックエンドがデフォルトシェルを起動
- バックエンドが `command` 省略をサポートしない場合:
  - フロントエンドで検出不能 → バックエンド修正が必要 → 本 task のスコープ外とし、暫定で `/bin/bash` をデフォルトにする
- 作成後に自動的に新セッションをアクティブにし、フォーカスを当てる

## 完了条件

- 「+」ボタンクリックでプレーンシェルが起動する
- 起動後に自動的にそのセッションがアクティブになる

## 検証方法

- 「+」ボタンでターミナルが起動し、シェルプロンプトが表示されることを確認

## 依存関係

- なし

## 成果物

- `src/features/terminal/TerminalSurface.tsx` (変更)
- `src/features/shell/CommandLauncher.tsx` (変更)
- `src/features/terminal/api.ts` (必要に応じて変更)
