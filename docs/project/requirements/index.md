# Requirements

## 役割

このディレクトリは、何を実現するかを管理する正本です。

## 置いてよいもの

- 目標
- 根拠
- 成功指標
- 対象範囲と対象外
- 制約や品質条件

## 置いてはいけないもの

- 実装手段の詳細
- テスト手順の詳細
- 一時的な作業メモ

## 命名規則

- `v<major>/REQ-<category>-<short-title>.md`

## 参照ルール

- 要求変更時は spec / validation / ADR への影響を確認する

## 参照

- `v0/index.md`: v0 / MVP の対象範囲と要求
- `v1/index.md`: v1 で対応する推奨機能と拡張要求
